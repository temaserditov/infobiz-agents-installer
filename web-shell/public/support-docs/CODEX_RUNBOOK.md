# Runbook для Codex

## Цель

Найти первопричину, сделать минимальное исправление и доказать, что WebShell и
каждый установленный gateway снова работают. Не маскировать проблему полной
переустановкой.

## 1. Определить среду

Соберите без вопросов к пользователю:

```bash
uname -a
uname -m
printf 'HOME=%s\n' "$HOME"
test -d "$HOME/InfobizAgents" && echo 'InfobizAgents: installed'
test -x "$HOME/.local/bin/hermes" && "$HOME/.local/bin/hermes" --version
test -x "$HOME/.hermes/hermes-agent/venv/bin/hermes" && "$HOME/.hermes/hermes-agent/venv/bin/hermes" --version
```

Классифицируйте установку:

- macOS: сервисы `launchd`, панель обычно доступна только локально.
- Linux/VPS: сервисы `systemd`, панель доступна по выданной установщиком ссылке.
- Windows ученика при VPS-сценарии: это только клиент. Hermes и его логи находятся
  на VPS.

Не просите пользователя устанавливать Python, Node.js, Homebrew или npm вручную.
Официальный setup Hermes и Infobiz-установщик управляют своими зависимостями сами.

## 2. Собрать факты без изменений

Найдите исполняемый файл Hermes:

```bash
HERMES="$HOME/.local/bin/hermes"
test -x "$HERMES" || HERMES="$HOME/.hermes/hermes-agent/venv/bin/hermes"
"$HERMES" doctor
"$HERMES" dump
```

Проверьте наличие профилей:

```bash
for profile in marketer copywriter designer tech; do
  test -d "$HOME/.hermes/profiles/$profile" && echo "$profile: present"
done
```

Проверьте сервисы по [SYSTEM_MAP.md](SYSTEM_MAP.md). Затем прочитайте:

```text
~/InfobizAgents/install.log
~/InfobizAgents/update.log
~/InfobizAgents/web-shell.out.log
~/InfobizAgents/web-shell.err.log
~/.hermes/logs/gateway.log
~/.hermes/profiles/<profile>/logs/gateway.log
```

Для логов используйте `tail`, `rg` и `hermes logs`; не выгружайте файлы целиком.
Ищите первую причинную ошибку перед каскадом повторных ошибок.

## 3. Определить масштаб

- Не работает WebShell, но Telegram отвечает: проблема панели или её сервиса.
- Не работает один агент: профиль, его токен, модель, сессия или отдельный gateway.
- Не работают все агенты: общая авторизация, Hermes runtime, сеть, диск или массовый
  перезапуск сервисов.
- Telegram не отвечает, но агент проходит локальный smoke test: токен, Telegram ID,
  доступ к `api.telegram.org` или gateway.
- Ошибка появилась во время установки: работайте с `install.log`; не диагностируйте
  её по состоянию частично созданного профиля как готовую установку.

## 4. Перед изменением

1. Проверьте актуальность официального Hermes и текущего Infobiz-апдейтера.
2. Опишите одним предложением найденную причину и минимальное исправление.
3. Сделайте резервную копию только файлов, которые будете менять.
4. Не копируйте `.env`, `auth.json` или сессии между профилями без доказанной причины.
5. Не меняйте `SOUL.md`, `IDENTITY.md`, `USER.md`, `MEMORY.md` и `skills/`, если
   проблема относится к runtime, сети, OAuth, Telegram или сервису.

## 5. Порядок ремонта

Используйте самый узкий подход:

1. Перезапустить только упавший сервис.
2. Исправить конкретное значение через WebShell или официальный Hermes CLI.
3. Запустить `hermes doctor --fix`, если `doctor` явно предлагает поддерживаемое исправление.
4. Выполнить актуальный апдейт Infobiz Agents, если установленный runtime или наш
   слой устарел.
5. Полная переустановка допустима только после резервной копии и доказательства,
   что более узкий ремонт невозможен.

Не редактируйте исходники внутри `~/.hermes/hermes-agent` вручную. Если нужен
совместимый патч, он должен быть воспроизводимо добавлен в репозиторий установщика,
проверен smoke-тестами и доставлен апдейтером.

## 6. Проверка после ремонта

Обязательно проверьте:

1. WebShell отвечает локально.
2. Все установленные сервисы находятся в состоянии running/active.
3. Каждый профиль видит корректное имя роли.
4. Для каждого подключённого Telegram-бота проходит короткое сообщение.
5. Ответ приходит один раз, полностью и на русском языке.
6. В свежем хвосте логов нет новых traceback, auth failure и циклических рестартов.

Не объявляйте задачу выполненной только потому, что команда завершилась с кодом 0.

## 7. Когда остановиться

Не угадывайте и не расходуйте повторно одноразовые токены, если:

- сервис выдачи курса отклоняет несколько свежих токенов;
- VPS не принимает реквизиты, выданные хостингом;
- официальный провайдер недоступен или не разрешает выбранную модель;
- исправление требует менять продуктовый патч, а не конкретную установку.

Соберите безопасный пакет по [SECURITY_AND_ESCALATION.md](SECURITY_AND_ESCALATION.md)
и эскалируйте с установленной причиной, а не со скриншотом терминала.
