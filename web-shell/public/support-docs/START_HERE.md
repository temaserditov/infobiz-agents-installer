# Поддержка Infobiz Agents

Эта документация предназначена для Codex, который помогает установить или
починить Infobiz Agents на Mac либо на VPS. Текущий продукт работает на Hermes.
Команды и пути OpenClaw к нему не применяются.

## Быстрый старт для пользователя

Откройте папку установки в Codex и напишите:

> Прочитай `web-shell/public/support-docs/CODEX_RUNBOOK.md`, проведи диагностику
> Infobiz Agents и исправь проблему. Сначала собери факты, не показывай секреты,
> не удаляй профили, память, скиллы и авторизацию. После исправления проверь все
> gateway и WebShell.

Стандартная папка установки:

```text
~/InfobizAgents
```

Если агенты находятся на VPS, сначала подключите Codex к серверу по SSH. Codex
должен выполнять диагностику на VPS, а не на локальном Windows-компьютере.

## Что читать

1. [CODEX_RUNBOOK.md](CODEX_RUNBOOK.md) — обязательный порядок диагностики и ремонта.
2. [SYSTEM_MAP.md](SYSTEM_MAP.md) — реальные пути, профили, сервисы и логи продукта.
3. [TROUBLESHOOTING.md](TROUBLESHOOTING.md) — решения по симптомам.
4. [SECURITY_AND_ESCALATION.md](SECURITY_AND_ESCALATION.md) — секреты, резервные копии и эскалация.
5. [OPENCLAW_SUPPORT_AUDIT.md](OPENCLAW_SUPPORT_AUDIT.md) — выводы из старых потоков; это не инструкция по ремонту Hermes.

## Главные правила

- Сначала диагностика, затем изменение.
- Для обновления продукта используется только актуальный установщик Infobiz
  Agents. Прямой `hermes update` не заменяет наш апдейтер.
- Официальный Hermes устанавливается первым. Наши профили, WebShell и точечные
  совместимые патчи накладываются поверх него.
- Нельзя удалять `~/.hermes`, переустанавливать всё или вручную менять исходники
  Hermes как первый способ лечения.
- Нельзя переносить команды, конфиги и пути из OpenClaw.
- Нельзя публиковать Telegram-токены, OAuth-данные, пароли VPS и ссылку WebShell
  с токеном доступа.
