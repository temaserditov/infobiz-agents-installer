# Безопасность и эскалация

## Что считается секретом

- пароль и приватный SSH-ключ VPS;
- одноразовый course token;
- Telegram Bot Token;
- Telegram ID пользователя, если он не нужен для диагностики;
- OAuth access/refresh tokens и содержимое `auth.json`;
- API-ключи, включая Groq;
- полная ссылка WebShell с параметром `token`;
- cookies, session-файлы и содержимое `.env`.

Секрет нельзя отправлять в чат, коммитить, вставлять в issue или показывать на
скриншоте. Для проверки достаточно сообщить, что значение существует, его длину,
маску последних четырёх символов либо результат авторизации.

## Диагностика вместо скриншотов

Предпочтительный порядок:

1. Локальный доступ Codex к установленной папке.
2. SSH-доступ Codex к VPS.
3. Редактированный support bundle WebShell.
4. Текст конкретной ошибки и последние связанные строки логов.
5. Скриншот только для проблем интерфейса.

Временный LAN-доступ к support bundle включается владельцем машины командой из
репозитория `support-infobiz-agents.sh`. После диагностики его необходимо выключить.
Ссылка с токеном не должна оставаться в публичном чате.

## Резервная копия перед ремонтом

Для изменения одного файла:

```bash
cp -p path/to/file "path/to/file.backup.$(date +%Y%m%d%H%M%S)"
```

Для официального Hermes используйте поддерживаемые snapshots/backup. Для полного
обновления продукта сначала убедитесь, что апдейтер сохраняет пользовательские
`.env`, auth, sessions, memory и локальные настройки.

## Что приложить к эскалации

- платформа и архитектура;
- Mac или VPS, а для VPS — дистрибутив и хостинг;
- этап: установка, OAuth, WebShell, gateway, Telegram, модель или обновление;
- точное время ошибки и профиль агента;
- первая причинная ошибка из лога;
- статусы сервисов;
- версия Hermes и версия/commit установщика;
- что уже проверено и что изменено;
- безопасный support bundle либо минимальный редактированный фрагмент.

Не отправляйте длинную историю проб. Нужны текущее состояние и воспроизводимый шаг.

## Границы автоматического ремонта

Codex не должен автоматически:

- сбрасывать пароль VPS или менять SSH-политику хостинга;
- расходовать новые одноразовые course tokens;
- менять тариф/доступность модели на стороне OpenAI;
- удалять профили, память или пользовательские документы;
- открывать WebShell всему интернету без токена;
- отключать firewall целиком;
- переносить OAuth или Telegram-секреты между людьми;
- делать постоянный патч в официальном Hermes только на одной машине.

Если проблема продуктовая, исправление вносится в установщик, проходит smoke-тесты,
а затем доставляется обычным апдейтом всем затронутым установкам.
